# Codex Prompt：检查 Obsidian Concepts 文件夹互链质量

你现在要扮演一个“Obsidian 知识库结构审计员”。请检查我当前 Obsidian vault 中 `concepts` 文件夹里的概念笔记互链是否合理、有意义，并输出一份结构化报告和修改建议。

## 背景

我的 Obsidian vault 大致分成两个模块：

1. `courses` 或类似名称的章节笔记模块  
   - 按教科书章节组织。
   - 单个文件通常包含较多文字、推导、公式、例题、上下文和学习过程。
   - 章节笔记主要承担“教材脉络”和“推导过程”的功能。

2. `concepts` 概念笔记模块  
   - 每个文件通常较短。
   - 主要描述章节笔记中涉及到的概念、定义、性质、辨析和使用场景。
   - 概念笔记应该形成一个有意义的概念网络，而不是简单的关键词高亮集合。

现在我希望你检查 `concepts` 文件夹中的概念文件互链是否合理，并给出优化建议。

## 核心目标

请重点判断：

- 当前概念之间的链接是否表达了真实的知识关系。
- 是否存在过度链接、无意义链接、机械链接或关键词高亮式链接。
- 是否存在应该链接但没有链接的概念关系。
- 是否存在概念文件过于孤立、过于碎片化或层级混乱的问题。
- 是否需要增加“相关概念”“前置概念”“容易混淆”“应用场景”等结构化区域。
- 是否有概念之间的关系需要用文字说明，而不是仅仅放一个 `[[link]]`。

请不要只统计链接数量。链接数量不是核心指标。核心指标是：链接是否帮助理解。

## 检查范围

优先检查：

- `concepts/` 文件夹下的所有 Markdown 文件。
- 这些文件之间的 Obsidian 内部链接，例如 `[[概念名]]`、`[[概念名|别名]]`。
- 每个概念文件中的出链、入链和上下文。

如果仓库中还有章节笔记文件夹，例如 `notes/`、`chapters/`、`textbook/`、`课程笔记/` 等，也可以辅助参考它们，但不要修改，也不要让章节笔记的结构替代对 `concepts` 文件夹的分析。

## 判断标准

请按照以下标准评估概念之间的链接。

### 一、值得保留或新增的链接

如果两个概念之间存在以下关系，通常值得链接：

1. 定义依赖  
   A 的定义需要 B 才能说清楚。  
   例如：导数依赖极限。

2. 推导依赖  
   A 的推导、证明或计算过程使用了 B。  
   例如：最小二乘法依赖正交投影。

3. 前置知识关系  
   理解 A 通常需要先理解 B。  
   例如：特征值通常需要先理解线性变换和矩阵。

4. 推广 / 特例关系  
   A 是 B 的特例，或者 B 是 A 的推广。  
   例如：欧氏空间是内积空间的特殊情形。

5. 对比 / 易混淆关系  
   A 和 B 容易混淆，或经常需要放在一起辨析。  
   例如：充分条件与必要条件；相关性与因果性。

6. 应用关系  
   A 经常用于理解、推导或应用 B。  
   例如：特征值用于稳定性分析。

7. 跨章节桥梁  
   两个概念在教材章节中距离较远，但实质上有重要联系。  
   这类链接尤其值得保留或新增。

### 二、可能需要删除或弱化的链接

如果链接属于以下情况，请标记为可疑：

1. 只是词语出现，并没有实际关系说明。
2. 链接目标不是理解当前概念所必要的。
3. 链接过多导致正文可读性下降。
4. 同一文件里反复链接同一个概念，但没有新增语义。
5. 链接放在句子中很突兀，只像关键词高亮。
6. 链接目标过于宽泛，例如几乎所有概念都链接到“数学”“函数”“定义”这类泛概念。
7. 链接指向的概念文件内容过薄，无法真正支持当前概念。

### 三、需要补充文字说明的链接

请特别检查这些情况：

- 两个概念确实有关，但当前文件只是简单写了 `[[概念A]]`，没有解释为什么有关。
- 多个概念被列在“相关概念”里，但没有分类或关系说明。
- 一个概念的出链很多，但读者无法判断这些链接分别是前置、应用、对比、推广还是例子。

这类链接不一定要删除，但通常需要补充一句关系说明。

例如不要只写：

```markdown
相关概念：[[特征值]]、[[特征向量]]、[[线性变换]]
```

更好的写法是：

```markdown
## 相关概念

- 前置概念：[[线性变换]] 是定义特征值和特征向量的背景。
- 配对概念：[[特征向量]] 是在线性变换作用下方向不变的向量，对应的伸缩比例就是 [[特征值]]。
```

## 你需要完成的任务

请分步骤完成以下工作。

### Step 1：扫描文件结构

请先识别 vault 中和任务相关的文件夹结构，尤其是：

- `concepts/` 文件夹的位置。
- 概念文件数量。
- 是否存在章节笔记文件夹。
- 是否存在命名约定，例如中文名、英文名、编号、前缀、别名等。

### Step 2：建立概念链接图

请解析 `concepts/` 下所有 Markdown 文件，提取：

- 每个概念文件的标题或文件名。
- 每个文件链接到哪些概念。
- 哪些概念链接回它。
- 哪些概念没有出链。
- 哪些概念没有入链。

- 哪些概念入链异常多。
- 哪些链接是断链或疑似拼写 / 命名不一致。

### Step 3：抽样阅读链接上下文

不要只做静态链接统计。请阅读链接附近的上下文，判断链接是否有语义。

优先检查：

- 出链最多的概念文件。
- 入链最多的概念文件。
- 完全孤立或几乎孤立的概念文件。
- 看起来像基础概念、核心概念或枢纽概念的文件。
- 包含大量 `[[...]]` 的文件。
- 包含“相关概念”“see also”“参考”“前置知识”等区域的文件。

### Step 4：分类评估链接质量

请把发现的问题分成以下几类：

1. 建议保留的高质量链接  
   说明为什么这些链接有意义。

2. 建议删除的低价值链接  
   说明为什么它们像关键词高亮或噪音。

3. 建议补充说明的链接  
   说明应该补什么关系文字。

4. 建议新增的缺失链接  
   说明为什么这些概念应该互相连接。

5. 建议合并、拆分或重命名的概念文件  
   针对过薄、重复、命名混乱或边界不清的概念文件提出建议。

6. 断链、重定向和别名问题  
   包括大小写、中文/英文混用、同义概念重复建档等。

### Step 5：提出 Obsidian 结构优化方案

请基于我的笔记结构提出建议，而不是泛泛而谈。

重点回答：

- 概念文件之间应该采用什么链接原则？
- 是否建议在每个概念文件末尾增加统一的“相关概念”区块？
- 如果增加，推荐什么模板？
- 哪些概念适合做 hub note / MOC？
- 哪些概念不适合继续碎片化？
- 哪些链接应该放在正文中，哪些链接应该放在“相关概念”区块中？

## 修改建议的风格

请尽量具体，不要泛泛地说“增加链接”或“减少链接”。

不好的建议：

```text
建议增加相关概念链接。
```

好的建议：

```text
在 `concepts/特征值.md` 的定义段落后新增一句：
“在线性变换视角下，[[特征向量]] 是方向不变的向量，而 [[特征值]] 描述该方向上的伸缩比例。”
这样可以把 `[[线性变换]]`、`[[特征向量]]` 和 `[[特征值]]` 的关系讲清楚，而不是只做关键词链接。
```
